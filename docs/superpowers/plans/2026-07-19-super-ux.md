# super-ux Repo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the public `ssheleg/super-ux` repo: a Claude Code plugin (skills `ux-scenarios` + `ux-audit`, commands `/ux-init` `/ux-update` `/ux-audit` `/ux-rule`), Cursor rules, templates, validator + CI, per the spec `docs/superpowers/specs/2026-07-19-super-ux-design.md`.

**Architecture:** Content repo in the plugin-marketplace layout (mirrors `ssheleg/task-pipeline`). `test/validate.py` is the test suite: written first, red while files are missing, green when the repo is complete and consistent. All contracts (file formats, verdicts, rule text) are locked in the spec — every authored file embeds them verbatim from spec §4–§7.

**Tech Stack:** Markdown + YAML front-matter, JSON manifests, Python 3 stdlib (validator), POSIX sh (installer), GitHub Actions.

## Global Constraints

- Repo language: English (README gets a closing Russian section).
- Version everywhere: `0.1.0` — `marketplace.json`, `plugin.json`, first CHANGELOG entry (validator enforces sync).
- License: MIT, holder `ssheleg`.
- Skill front-matter: `name` matches directory name; `description` states triggers.
- Cursor rules: `.mdc` with `alwaysApply` + `description` front-matter (per Cursor docs, fetched 2026-07-19).
- No placeholders anywhere: no TBD/TODO in shipped files.
- The shell in this session is degraded (minutes per spawn): batch ALL git/validation commands into single background Bash scripts; author files with the Write tool only.
